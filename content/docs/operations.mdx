前端文档和地图应用可部署到 Netlify，空间数据库、切片服务和 GeoServer 建议部署在受控云环境或内网 Kubernetes 集群。

## 发布检查清单

- 确认生产环境 API 域名、地图 Token 和图层配置不写入源码。
- 确认 PostGIS 扩展、空间索引和慢查询日志已启用。
- 为切片、导出文件和影像配置对象存储生命周期策略。
- 配置服务健康检查、错误告警、磁盘容量告警和数据库备份。
- 将数据发布、样式发布和应用发布拆分，避免一次变更影响全部链路。

## 当前文档部署

当前技术文档是 Fumadocs + Next.js 站点，已连接 GitHub 与 Netlify；推送到 `main` 后会自动触发生产部署。
