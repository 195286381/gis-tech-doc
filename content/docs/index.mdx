## 概览

本文档定义一套面向生产环境的 GIS 技术方案，覆盖空间数据接入、坐标转换、地图切片、空间查询服务、性能优化、安全控制与运维发布。

| 项目 | 说明 |
| --- | --- |
| 适用场景 | 政务、园区、物流、资产管理、自然资源 |
| 核心栈 | PostGIS、GeoServer、MapLibre、Netlify |
| 文档版本 | v1.2 / Fumadocs |

## 文档结构

- **架构与坐标**：定义系统分层、核心组件、坐标基准和投影转换原则。
- **数据工程**：定义空间数据模型、质量门禁、入库流程和切片策略。
- **接口与生产**：定义空间查询 API、性能基线、安全权限、运维发布和验收清单。

## 发布方式

当前站点使用 Fumadocs 构建，GitHub 仓库连接到 Netlify。推送到 `main` 分支后，Netlify 会自动构建并发布。

```bash
git add .
git commit -m "Update GIS docs"
git push
```
